How a Quote is Minted

From boot to on-chain verification in five steps.

01

TEE Boot

The CPU creates an encrypted memory region. Every byte of code loaded is measured — hashed into tamper-evident registers that can never be rolled back.

02

Compute Value X

bountynet-shim walks every file in the runner directory, hashes them deterministically, and produces a single 48-byte fingerprint. Same code = same X on any platform.

03

Bind & Sign

Value X and a fresh signing key are bound into the TEE's report data. The hardware signs a quote proving this key was generated inside a genuine enclave.

04

Wrap in UnifiedQuote

The platform-specific evidence gets wrapped into a single format with an ed25519 signature. ~180 bytes on-chain, full quote off-chain and hash-linked.

05

Serve & Verify

The runner serves /attest for remote verification while executing your jobs. Anyone can verify Layer 1 (signature) or Layer 2 (full hardware chain).
